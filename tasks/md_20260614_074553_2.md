# 需求文档

## 1. 应用概述

### 1.1 应用名称
开源应用商店

### 1.2 应用描述
一款面向移动端的开源应用商店APP，通过GitHub API自动发现并聚合包含可安装文件的开源项目，为用户提供跨平台应用的发现、搜索、收藏与下载服务。支持Android、iOS、Web三端运行。

## 2. 用户与使用场景

### 2.1 目标用户
- 开源软件爱好者
- 开发者
- 需要寻找跨平台开源应用的普通用户

### 2.2 核心使用场景
- 用户希望发现热门或最新的开源应用
- 用户需要按分类或关键词搜索特定类型的开源应用
- 用户需要追踪已关注应用的版本更新
- 用户希望收藏感兴趣的应用以便后续查看

## 3. 页面结构与功能说明

### 3.1 页面结构

```
开源应用商店APP
├── 首页 Tab
├── 发现页 Tab
```

## 4. 业务规则与逻辑

### 4.1 自动发现机制

- 通过GitHub Search API自动发现包含APK、EXE、DMG等可安装文件的公共仓库
- 无需开发者手动提交应用
- 定期执行自动发现任务，更新应用库

### 4.2 平台识别规则

- 根据仓库Release中的资产文件扩展名识别支持的平台
  - .apk → Android
  - .ipa → iOS
  - .dmg → macOS
  - .exe/.msi → Windows
- 在应用列表和详情页中标识支持的平台

### 4.3 智能分类规则

- 基于repository.topics自动提取分类标签
- 基于repository.language识别编程语言分类
- 基于README关键词匹配预设的分类映射规则
- 分类包括但不限于：开发工具、效率工具、媒体、游戏、安全

### 4.4 更新追踪规则

- 用户可配置更新检查频率：3小时、6小时、12小时、24小时
- 后台定时检查已收藏应用的新版本
- 发现新版本后在应用内显示更新通知
- 通知内容包括：应用名称、新版本号、更新时间

### 4.5 数据缓存规则

- 将GitHub API获取的应用元数据缓存到后端数据库
- 缓存内容包括：应用基础信息、README内容、版本历史、Star/Fork数
- 优先从缓存读取数据，减少GitHub API调用频率
- 定期更新缓存数据

### 4.6 数据质量过滤规则

- 设置Star数最低阈值，过滤低质量项目
- 过滤最近更新时间超过一定期限的项目
- 过滤缺少README或文档不全的项目

### 4.7 API调用配额管理

- 匿名用户：60次/小时
- GitHub登录用户（提供PAT）：5000次/小时
- 通过后端Edge Function调用GitHub API，保护Token安全
- 用户提供的Token通过Edge Function调用时作为Authorization header传递
- 优先使用缓存数据，减少重复调用

### 4.8 真实数据获取规则

- App首次打开时自动触发Edge Function从GitHub拉取真实数据
- 应用卡片展示真实的GitHub仓库数据，应用图标使用owner_avatar_url
- 无图标时降级为字母头像（取应用名称首字母）
- 搜索时实时调用Edge Function从GitHub搜索并缓存结果

### 4.9 Token安全存储规则

- 用户输入的GitHub Personal Access Token使用expo-secure-store安全存储
- Token仅在调用Edge Function时作为Authorization header传递
- 用户可随时清除已保存的Token

### 4.10 下载跳转规则

- 所有「下载」按钮点击后直接跳转GitHub releases页面
- 跳转链接格式：https://github.com/{owner}/{repo}/releases
- 版本历史中的「下载」按钮跳转对应版本的releases页面

### 4.11 图片懒加载规则

- 应用列表中的应用图标采用懒加载策略
- 仅加载可视区域内的图片
- 滚动时动态加载进入可视区域的图片

### 4.12 列表虚拟化规则

- 应用列表采用虚拟化技术
- 仅渲染可视区域内的列表项
- 提升长列表滚动性能

### 4.13 搜索历史记录规则

- 本地存储用户最近20条搜索记录
- 按时间倒序展示
- 用户可清空全部历史记录

### 4.14 下载历史记录规则

- 本地存储用户的下载记录
- 记录内容包括：应用名称、版本号、下载时间、文件大小
- 用户可清空全部下载记录

### 4.15 收藏分组规则

- 用户可创建自定义分组
- 每个应用可归属于一个或多个分组
- 默认分组为「全部收藏」

### 4.16 MD5校验规则

- 应用详情页展示安装包的MD5校验值
- 用户下载后可自行校验文件完整性

### 4.17 动画规则

- 页面切换时播放过渡动画
- 应用卡片进入可视区域时播放渐入动画
- 下载进度条播放平滑动画
- 骨架屏加载时播放闪烁动画

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 网络连接失败 | 提示用户网络异常，支持重试操作 |
| GitHub API调用超出配额 | 提示用户稍后再试，或引导用户在「我的」Tab输入GitHub Token提升配额 |
| 应用详情页README内容为空 | 显示「暂无README文档」提示 |
| 应用无可用安装包 | 显示「暂无可用下载」提示 |
| 用户设备平台无对应安装包 | 显示「当前平台暂不支持」提示，并展示其他平台的安装包 |
| 搜索无结果 | 显示「未找到相关应用」提示 |
| 收藏页为空 | 显示「暂无收藏应用」提示 |
| 后端数据库缓存失败 | 降级为直接调用GitHub API |
| 版本历史为空 | 显示「暂无版本历史」提示 |
| Token输入格式错误 | 提示用户「Token格式错误，请检查后重新输入」 |
| Token权限不足 | 提示用户「Token权限不足，请检查Token权限设置」 |
| 首次加载数据失败 | 显示「数据加载失败，请检查网络后重试」，提供重试按钮 |
| owner_avatar_url无法加载 | 降级为字母头像（取应用名称首字母） |
| 搜索历史记录为空 | 显示「暂无搜索历史」提示 |
| 下载历史记录为空 | 显示「暂无下载记录」提示 |
| 截图/预览图无法加载 | 显示「暂无预览图」提示 |
| 相似应用推荐为空 | 隐藏相似应用推荐区块 |
| 贡献者信息为空 | 隐藏贡献者头像展示区 |
| 导出收藏列表失败 | 提示用户「导出失败，请重试」 |
| 批量下载时部分应用无法下载 | 跳过无法下载的应用，继续下载其他应用 |
| MD5校验值缺失 | 显示「暂无校验值」提示 |

## 6. 验收标准

1. 用户打开APP，首页展示浅灰色背景（#F2F3F5），顶部白色圆角搜索框和通知铃铛，Banner轮播区自动轮播，推荐应用区横向滚动展示方形圆角图标网格（带版本标签），应用列表展示白色圆角卡片（带轻柔阴影），应用图标使用owner_avatar_url，无图标时显示字母头像
2. 用户在首页点击某个应用卡片，进入应用详情页，查看截图/预览图、README内容、贡献者头像、版本历史、相似应用推荐
3. 用户在应用详情页点击「下载」按钮（白底蓝边框#1677FF，圆角20px胶囊形），跳转GitHub releases页面进行下载，下载记录自动保存到下载历史
4. 用户点击收藏按钮，将应用添加到收藏页，创建自定义分组，为应用添加标签，导出收藏列表为JSON文件
5. 用户进入搜索页，输入关键词搜索应用，查看搜索历史记录和热门搜索标签，实时搜索建议自动展示，搜索结果支持分类筛选
6. 用户进入发现页，使用平台筛选器和排序选项，浏览瀑布流布局的分类应用列表
7. 用户进入「我的」Tab，输入GitHub Personal Access Token并保存，查看API配额状态从60次/小时提升至5000次/小时，查看收藏统计信息
8. 用户进入下载历史页，查看下载记录，使用批量下载功能，管理下载队列
